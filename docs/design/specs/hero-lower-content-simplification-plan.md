# Hero Lower Content Simplification Plan

## 目的

Heroの主役を3Dキューブと右側の選択パネルに絞る。

現在は下部カード、選択中パネル、各セクション本文が続いているため、Heroの体験が少し散らばって見える。

このブランチでは、Hero下部の情報量を一度減らし、キューブ操作だけで導線が成立するか確認する。

## 方針

-   キューブを最初の画面の主役にする
-   右側パネルを選択中セクションへの導線として使う
-   下部カードから下は一旦削除する
-   Projects / Skills / About / Contact の詳細表現は後続ブランチで再設計する
-   削除後も `sections` のデータ構造は必要な範囲で残す

## 削除対象

### 下部カードナビ

`Projects / Skills / About / Contact` の4カード。

現状ではキューブと役割が重複しているため、一旦削除する。

### 選択中パネル

カード下に表示している選択中セクションの詳細パネル。

右側パネルが同じ役割を持ち始めているため、一旦削除する。

### Hero直下の各コンテンツセクション

`Projects`, `Skills`, `About`, `Contact` の本文セクション。

このブランチでは詳細コンテンツを作り込まず、Heroの導線整理に集中する。

## 残すもの

### Hero

-   Header
-   3Dキューブ
-   H1
-   リード文
-   View Projects / About Me ボタン
-   右側の選択パネル

### セクションデータ

右側パネルの表示に必要なため、`sections` のデータは残す。

ただし、下部カードや本文セクションだけに使っている値があれば整理する。

## 右側パネルの役割

右側パネルは操作説明ではなく、選択中セクションのアクションとして扱う。

例:

``` text
SELECTED
Projects

WordPress制作からNext.jsを使ったWebアプリまで...

Open Projects
```

CTAは後続ブランチで詳細画面やモーダルへつなげる前提にする。

このブランチでは、リンク先が同一ページ内アンカーのままでもよい。

## 実装順

### Step 1. HeroShellから下部カードを削除

`sectionNav` を削除する。

キューブ、右側パネル、Hero内ボタンの動作は維持する。

### Step 2. 選択中パネルを削除

`selectedPanelWrap` と `selectedPanel` の表示を削除する。

削除後、不要になった CSS を整理する。

### Step 3. ページ下部セクションを削除または非表示

`page.tsx` 側の Projects / Skills / About / Contact セクションを確認し、このブランチでは削除または非表示にする。

まずは削除対象を最小にし、データ構造を壊しすぎない。

### Step 4. Heroの余白を再調整

下部カードがなくなるため、Heroの高さ、余白、次のコンテンツとの関係を見直す。

最初の画面でキューブと右側パネルが気持ちよく収まるようにする。

### Step 5. build / lint

変更後に以下を確認する。

``` bash
npm run build
npm run lint
```

## 注意点

-   キューブ操作は壊さない
-   `activeLabel` と右側パネルの連動は維持する
-   H1と右側パネルの役割が重複しすぎないようにする
-   CSS削除は慎重に行い、後続で使いそうな共通スタイルは残す
-   モバイル表示で余白が大きく空きすぎないようにする

## ゴール

Heroを、3Dキューブを中心にしたシンプルなファーストビューへ整理する。

下部に情報を並べるのではなく、キューブと右側パネルから次の体験へ進む形を作る。
