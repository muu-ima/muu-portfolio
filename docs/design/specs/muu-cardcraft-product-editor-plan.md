# muu-cardcraft Product Editor Plan

## 目的

`muu-cardcraft` の名刺作成エディターを、スマホケースやTシャツなどの商品デザインエディターへ拡張できるか検討する。

この資料では、第一弾をスマホケース、第二弾をTシャツとして扱う。

既存の2D編集エンジンを活かしながら、完成イメージは3Dモデルで確認できる体験を目指す。

## コンセプト

2Dで作り、3Dで確認する商品エディター。

-   テキストや画像の配置は2Dキャンバスで行う
-   変更内容は3Dモデルのプレビューへ反映する
-   3Dモデルはドラッグで回転できる
-   書き出しは印刷用PNGを基本にする

3D上で直接文字や画像を編集するのではなく、まずは2D編集と3Dプレビューを分ける。

これにより、操作性を安定させながら、商品としての見え方も確認できる。

## 既存資産

`muu-cardcraft` には、商品エディターへ転用できる資産がすでにある。

### 編集データ

テキスト、画像、座標、レイヤー順、表裏の概念を持っている。

スマホケースやTシャツでも、基本的にはこの構造を使える。

### 編集キャンバス

`CardSurface` は、指定した幅と高さの中にテキストや画像を配置する構造になっている。

名刺サイズをスマホケース用、Tシャツ用の印刷エリアへ差し替えれば、編集体験を流用できる。

### 画像書き出し

`ExportSurface` と `html-to-image` によって、編集面をPNG/JPEGとして書き出せる。

この書き出し画像を3Dモデルのテクスチャとして使う。

### 3D表現

現在のポートフォリオで作った3Dキューブの経験を活かせる。

-   React Three Fiber の構成
-   ライトとマテリアルの調整
-   ドラッグ回転
-   選択中状態の見せ方
-   3Dを主役にしたUI構成

キューブをそのまま変形するより、商品ごとの3Dモデルへ置き換える方が現実的。

## 第一弾: スマホケース

スマホケースは、`muu-cardcraft` からの転用に最も向いている。

理由は、印刷面が比較的平面に近く、名刺エディターの構造と相性が良いため。

### 必要な変更

-   キャンバス比率をスマホケースに変更する
-   角丸の外形を表現する
-   カメラ穴をガイドとして表示する
-   セーフエリアを定義する
-   端に近い要素は注意表示できるようにする
-   3Dスマホケースモデルへ印刷画像を貼る

### 編集キャンバス

2Dキャンバスは、スマホケースの印刷面として扱う。

表示するガイド:

-   外形ライン
-   カメラ穴
-   セーフエリア
-   中央線
-   印刷可能範囲

名刺のような完全な長方形ではなく、スマホケース特有の形状をガイドで伝える。

### 3Dプレビュー

メインプレビューは3Dスマホケースにする。

-   正面表示
-   斜め表示
-   ドラッグ回転
-   ケース色の切り替え
-   デザイン画像の即時反映

初期MVPでは、カメラ穴や側面まで完璧にマッピングしなくてよい。

まずは正面の印刷エリアに画像が自然に貼られて見えることを優先する。

### MVP

最初の実装範囲。

-   スマホケース用キャンバスサイズを追加する
-   カメラ穴とセーフエリアのガイドを表示する
-   テキスト追加、画像追加、移動、リサイズを使えるようにする
-   編集面をPNGとして書き出す
-   PNGを3Dスマホケースの正面に貼る
-   3Dモデルをドラッグで回転できるようにする

### 後続改善

-   ケース色の選択
-   機種選択
-   カメラ穴の形状差分
-   側面への回り込み表現
-   商品注文プレビュー
-   保存済みデザインの再編集

## 第二弾: Tシャツ

Tシャツは魅力が大きい一方で、スマホケースより難易度が高い。

布のしわ、体の曲面、サイズ差、印刷位置の見え方を考える必要がある。

### 必要な変更

-   胸元の印刷エリアを定義する
-   Tシャツ色を選択できるようにする
-   2Dキャンバスの比率を印刷範囲に合わせる
-   書き出し画像をTシャツモデルへ貼る
-   布の曲面に対して自然に見えるように調整する

### MVP

最初は高度な布シミュレーションを入れない。

-   正面Tシャツモデルを用意する
-   胸元の平面に近いエリアへデザインを貼る
-   色違いを切り替えられるようにする
-   3D回転で仕上がりを確認できるようにする

### 後続改善

-   デザインの布へのなじみ
-   影やしわに合わせた合成
-   サイズごとの印刷位置
-   背面プリント
-   袖プリント

## 共通アーキテクチャ

商品ごとにエディターを作り直さず、共通の編集エンジンへ寄せる。

``` text
Editor Data
  - text blocks
  - image layers
  - layout
  - product type
  - print area

2D Product Canvas
  - product guide
  - safe area
  - editable layers

Export Texture
  - PNG
  - transparent background option
  - fixed print resolution

3D Product Preview
  - model
  - material
  - texture mapping
  - rotate / zoom
```

## 商品タイプ設計

商品ごとの差分を設定として持つ。

``` ts
type ProductType = "business-card" | "phone-case" | "t-shirt";

type ProductPrintArea = {
  width: number;
  height: number;
  safeInset: number;
  bleed?: number;
  guide?: "card" | "phone-case" | "t-shirt";
};
```

名刺、スマホケース、Tシャツを同じ編集フローで扱えるようにする。

## UI方針

### デスクトップ

-   左: 2D編集キャンバス
-   右: 3D商品プレビュー
-   横または下: ツールパネル

3Dプレビューを常に見える位置に置き、編集結果が商品に反映される感覚を強くする。

### モバイル

モバイルでは、2D編集と3Dプレビューを同時表示しすぎない。

-   編集タブ
-   プレビュータブ
-   書き出しタブ

3Dモデルのドラッグ操作はスクロールと衝突しやすいため、モバイルではプレビュー画面内だけで操作できるようにする。

## ポートフォリオでの見せ方

この構想は、ポートフォリオの強い制作実績になる。

単なるデザインツールではなく、

-   既存エディターの再設計
-   商品ごとの印刷領域設計
-   2D編集と3Dプレビューの連携
-   Next.js / TypeScript / 3D UI の組み合わせ

を見せられる。

作品説明では、次のように伝える。

``` text
名刺作成エディター muu-cardcraft をベースに、
スマホケースやTシャツなどの商品デザインへ展開できる
3Dプレビュー付きエディターとして再設計。
```

## 実装順

### Step 1. スマホケース用の商品設定を作る

名刺サイズとは別に、スマホケース用のキャンバスサイズとガイド情報を定義する。

### Step 2. 2Dキャンバスをスマホケース表示にする

角丸、カメラ穴、セーフエリアを表示する。

既存のテキスト・画像編集機能はできるだけ流用する。

### Step 3. PNG書き出しを商品テクスチャ化する

編集面をPNG化し、3Dモデルのテクスチャとして使える形にする。

### Step 4. 3Dスマホケースプレビューを作る

最初は簡易モデルでよい。

角丸の板に厚みを持たせ、正面にテクスチャを貼る。

### Step 5. 2D編集と3Dプレビューを連動する

編集内容の変更後に、プレビュー用テクスチャを更新する。

最初は手動更新でもよいが、最終的には自動反映を目指す。

### Step 6. Tシャツへ展開する

スマホケースで共通設計が固まった後、Tシャツ用の商品設定と3Dモデルを追加する。

## 注意点

-   最初から3D上で直接編集しない
-   スマホケースを先に作り、Tシャツは後続にする
-   商品ごとの印刷可能範囲を曖昧にしない
-   モバイルでは3D操作とページスクロールの衝突を避ける
-   書き出し画像の解像度を商品ごとに管理する
-   3Dモデルの見た目に寄せすぎて、編集機能を複雑にしすぎない

## ゴール

`muu-cardcraft` を、名刺だけで終わらない商品デザインエディターへ拡張する。

第一弾としてスマホケースを作り、2D編集と3Dプレビューの基本構造を確立する。

その後、Tシャツなどのより表現力が必要な商品へ展開する。
