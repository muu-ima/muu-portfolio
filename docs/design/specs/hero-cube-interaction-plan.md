# Hero Cube Interaction Plan

## 目的

Heroの3Dキューブを、単なる装飾ではなく「触って遊べるナビゲーション」にする。

最初の画面で印象を残しつつ、Projects / Skills / About / Contact へ自然に誘導することを目指す。

## コンセプト

キューブはポートフォリオの入口として扱う。

-   ドラッグして回せる
-   面をクリックすると正面を向く
-   正面の面をもう一度クリックすると画面が切り替わる
-   操作しないときはゆっくり浮遊する

「3Dを見せる」よりも、「サイトを触りたくなる」体験を優先する。

## 基本操作

### 1. ドラッグで回転

ユーザーがキューブ全体をドラッグして回せるようにする。

-   横ドラッグで `rotation.y` を動かす
-   縦ドラッグは控えめに `rotation.x` へ反映する
-   ドラッグ中は自動回転や面への吸着を止める
-   離した直後は少し慣性を残す

縦方向は動きすぎると面のラベルが読みづらくなるため、角度制限を入れる。

### 2. 面クリックで正面化

面をクリックすると、その面が正面を向く。

-   Projects
-   Skills
-   About
-   Contact

クリック直後は画面遷移しない。

まず「選択した面を正面に向ける」ことをユーザーに見せる。

### 3. 正面の面をもう一度クリック

すでに正面を向いている面をもう一度クリックしたら、該当コンテンツへ切り替える。

切り替え時は、急に内容を変えずにフェードを使う。

-   キューブまたはHero全体を少し暗くする
-   選択面の発光を強める
-   下部/右側のコンテンツをフェードアウト
-   対応セクションをフェードイン

## 状態設計

必要になりそうな状態。

``` ts
type CubeInteractionState =
  | "idle"
  | "dragging"
  | "snapping"
  | "selected"
  | "transitioning";
```

### idle

通常状態。

キューブはゆっくり浮遊し、軽く動く。

### dragging

ユーザーがドラッグしている状態。

自動回転と吸着は止める。

### snapping

ドラッグ終了後、最も近い面へ吸着している状態。

### selected

面が正面を向いている状態。

同じ面をもう一度クリックすると `transitioning` へ進む。

### transitioning

画面切り替え中。

フェードや発光演出を行い、完了後に表示コンテンツを切り替える。

## 体験ルール

-   操作できることが見た目で分かるようにする
-   面ラベルは常に読める範囲に保つ
-   動きは気持ちよく、速すぎない
-   クリックとドラッグが衝突しないようにする
-   モバイルではドラッグしやすいサイズを確保する
-   キーボード操作も後で追加できる構造にする

## 実装順

### Step 1. 面クリックで正面を向く

現在のクリック操作を拡張し、クリックされた面が正面を向くようにする。

この段階では画面遷移はまだ入れない。

### Step 2. 同じ面の二度目クリック

正面を向いている面をもう一度クリックしたときだけ、切り替えの準備状態に入る。

まずは `console` や状態変更だけで確認してもよい。

### Step 3. フェード切り替え

Hero内の説明パネルや下部カードをフェードさせ、選択コンテンツへ切り替える。

### Step 4. ドラッグ回転

キューブ全体をドラッグで回せるようにする。

ドラッグ終了後は近い面へ吸着する。

### Step 5. 慣性と細かい気持ちよさ

ドラッグ終了時に少しだけ慣性を残す。

ただし、最終的には面へ吸着して迷子にならないようにする。

## 注意点

-   キューブが大きく見切れる問題を再発させない
-   台座との位置関係を保つ
-   右側コンテンツは今後別表現に変える可能性がある
-   そのため、キューブ単体で成立する構図を優先する
-   操作説明は最小限にし、体験で理解できるようにする

## ゴール

ユーザーが最初に触った瞬間、

「この人はUIと体験まで考えて作れる」

と伝わるHeroにする。
